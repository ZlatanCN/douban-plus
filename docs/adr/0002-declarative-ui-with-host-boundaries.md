# Declarative UI with host integration boundaries

douban-plus will preserve visual and behavioural compatibility while moving every owned UI update into Preact state and rendering. All owned dialogs render from a root-level modal layer, rather than from temporary or local Preact roots. Extractors are read-only over the connected Douban document; any normalization operates on detached data or cloned nodes. Direct DOM and browser-platform operations are allowed only at explicit host integration boundaries, where they read or interoperate with Douban's document or provide unavoidable platform lifecycle control; the sole owned-node adoption exception is a verified Douban login iframe mounted into the root-level login modal. Static styles belong in the stylesheet manifest, and external rich HTML is sanitized before controlled Preact HTML rendering. Fixed-fixture pixel baselines enforce visual compatibility while real-Douban E2E verifies behaviour.
